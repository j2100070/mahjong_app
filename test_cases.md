# 麻雀点数計算エンジン — 網羅的テストケース

**対象:** FR-20（点数計算エンジンの検証）  
**Date:** 2026-05-04

---

## 凡例

- **手牌表記:** 1m=一萬, 9p=九筒, 5s=五索, 東=東, 白=白 等
- **赤ドラ:** 0m=赤5萬, 0p=赤5筒, 0s=赤5索
- **副露:** [ポン:白白白] [チー:1m2m3m] [明カン:9p9p9p9p] [暗カン:發發發發]
- **期待値:** 子ロン点数を基準（ツモ・親の場合は別途記載）
- **アガリ牌入力方針:** テストでは全ケースにアガリ牌を明記する。実アプリでは以下の2方式を想定：
  - 方式A: 14枚入力 → 1枚を選択してアガリ牌に指定
  - 方式B: 13枚入力 → アガリ牌1枚を別途手動入力

## TC-01: 基本的な1翻の役

### TC-01-01: リーチのみ（40符1翻）
- 手牌: 5p6p7p 2s3s4s 6s7s8s 7m8m9m 9p9p（アガリ牌: 7m ロン、辺張待ち）
- 条件: 面前、リーチ、子
- 期待: 役=[リーチ], 符=**40**（基本20+メンゼンロン10+辺張2=32→**切り上げ40**）, 翻=1, 点数=**1,300**
- 備考: 平和不成立（辺張待ち 8m9m→7m）。リーチのみで30符は不可能（30符条件=平和条件と同一のため）

### TC-01-02: 平和ロン（30符1翻）
- 手牌: 1m2m3m 4m5m6m 7p8p9p 2s3s4s 5s5s（アガリ牌: 4s ロン、両面待ち）
- 条件: 面前、子、雀頭は役牌でない
- 期待: 役=[平和], 符=30, 翻=1, 点数=1,000

### TC-01-03: 平和ツモ（20符2翻 = 平和+ツモ）
- 手牌: 1m2m3m 4m5m6m 7p8p9p 2s3s4s 5s5s（アガリ牌: 4s ツモ、両面待ち）
- 条件: 面前、子
- 期待: 役=[平和, ツモ], 符=20, 翻=2, 点数=子各400/親700

### TC-01-04: タンヤオ面前ロン（40符1翻）
- 手牌: 2m3m4m 5p6p7p 3s4s5s 2p2p 6s7s8s（アガリ牌: 7s ロン、嵌張待ち）
- 条件: 面前、子、ドラ表示牌なし

- 備考: 嵌張待ちのため平和不成立。タンヤオのみで検証

### TC-01-05: クイタン（副露タンヤオ、30符1翻）

## TC-17: 数え役満（FR-13b）

### TC-17-01: 13翻=数え役満（子ロン32,000）
- 手牌: 1m2m3m 4m5m6m 7m8m9m 白白白 中中（アガリ牌: 中 ロン）
- 条件: 面前、リーチ、一発、子
- ドラ表示牌: 8m（ドラ=9m→1枚）、裏ドラ表示牌: 6m（裏ドラ=7m→1枚）
- 期待: 役=[リーチ,一発,一気通貫,ホンイツ,役牌:白,ドラ1,裏ドラ1]
- 翻計算: リーチ1+一発1+一気通貫2+ホンイツ3+白1+ドラ1+裏ドラ1=**10翻**→倍満
- 備考: 10翻では13翻に届かない。数え役満のテストにはさらにドラが必要。裏ドラ3枚追加で計13翻→数え役満
- 裏ドラ表示牌追加: 6m,5m,4m（裏ドラ=7m,6m,5m→合計3枚）
- 期待: 翻=13（リーチ1+一発1+一気通貫2+ホンイツ3+白1+ドラ1+裏ドラ4）, 点数=**32,000**（数え役満）

### TC-17-02: 12翻=三倍満（数え役満にならない）
- 手牌: 1m2m3m 4m5m6m 7m8m9m 白白白 中中（アガリ牌: 中 ロン）
- 条件: 面前、リーチ、子、他家カン1回

- 裏ドラ表示牌: 中（裏ドラ=白→3枚）、カン裏ドラ表示牌: 8m（裏ドラ=9m→1枚）
- 期待: 役=[リーチ,一気通貫,ホンイツ,役牌:白,ドラ1,裏ドラ4]
- 翻計算: リーチ1+一気通貫2+ホンイツ3+白1+ドラ1+裏ドラ4=**12翻**
- 期待: 点数=**24,000**（三倍満）※12翻は数え役満にならない

### TC-17-03: 数え役満の境界テスト（ドラ大量）
- 手牌: 2m3m4m 0p6p7p 3s4s0s 8m8m8m 2p2p（アガリ牌: 4m ロン）
