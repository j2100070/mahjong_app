## 1. Introduction / はじめに

### 1.1 Purpose / 目的

本アプリケーションは、麻雀のアガリ手牌・場の情報を入力し、点数を自動計算するWebアプリである。入力方法として**カメラによる牌画像認識**と**スワイプによる手動選択**の2つを提供する。

### 1.2 Scope / 範囲

- 日本麻雀（リーチ麻雀）のルールに準拠
- モバイル・デスクトップの両ブラウザで動作
- オフラインでも基本機能が利用可能

### 1.3 Stakeholders / ステークホルダー

| ステークホルダー | 説明 |
|---|---|
| 一般ユーザー | 麻雀プレイヤー（初心者〜上級者） |
| 開発者 | アプリの開発・保守担当者 |

## 4. System Architecture / システムアーキテクチャ

### 4.1 技術スタック

```
┌──────────────────────────────────────────┐
│         Frontend (Browser / React)       │
│                                          │
│  ┌────────────┐  ┌─────────────────┐    │
│  │ UI Layer   │  │ Camera Module   │    │
│  │ React+Vite │  │ getUserMedia API│    │
│  └─────┬──────┘  └────────┬────────┘    │
│        │                  │              │
│  ┌─────▼──────────────────▼────────┐    │
│  │      Tile Recognition Engine    │    │
│  │      TensorFlow.js / ONNX.js   │    │
│  └─────────────┬───────────────────┘    │
│                │                         │
│  ┌─────────────▼───────────────────┐    │
│  │     Scoring Engine (JS/TS)      │    │
│  │  ・役判定  ・符計算  ・点数算出    │    │
│  └─────────────────────────────────┘    │
└──────────────────────────────────────────┘
```

- **フロントエンドのみ**で完結（サーバー不要）
- **UI: React + Vite**（コンポーネント指向で牌選択UI・結果表示を管理）
- カメラ認識: TensorFlow.js またはONNX.js
- 点数計算ロジック: 純JavaScript/TypeScript

### 4.3 技術的考察: ReactとTensorFlow.jsの共存（v0.2対応）

**結論: Reactでv0.2（カメラ認識）を実装することに技術的な問題はない。**

| 懸念事項 | 評価 | 対策 |
|---|---|---|
| React再レンダリングとカメラストリーム競合 | 低リスク | `useRef`でvideoエレメント管理、`useEffect`でストリーム制御 |
| TensorFlow.jsとReactの互換性 | 問題なし | TensorFlow.jsはDOM非依存。推論はWeb Worker化可能 |
| バンドルサイズ増加 | 中リスク | TF.jsモデルは遅延読込（lazy load）で対応 |
| カメラ映像のリアルタイム処理 | 低リスク | `requestAnimationFrame`をReact外で管理し描画パフォーマンス維持 |
| PWA化との相性 | 問題なし | Vite PWAプラグインでService Worker自動生成 |

**推奨アーキテクチャ:**
- v0.1: React + Vite で手動入力UI・点数計算エンジンを構築
- v0.2: カメラモジュールをReactコンポーネント(`<CameraScanner />`)として追加。TF.jsの推論処理は`useRef` + カスタムフックで管理し、Reactの再レンダリングサイクルから分離する

### 4.2 主要コンポーネント

| コンポーネント | 説明 |
|---|---|
| Tile Selector UI | 牌をスワイプ/タップで選択するUI |
| Camera Module | カメラ映像から牌を認識するモジュール |
| Hand Validator | 手牌が有効なアガリ形かを判定 |
| Yaku Detector | 成立している役を判定 |
| Fu Calculator | 符数を計算 |
| Score Calculator | 翻・符から最終点数を算出 |

> **【七対子の符計算に関する特記事項】**
> 七対子は通常の4メンツ1雀頭の形ではなく7つの対子で構成されるため、通常の符計算（メンツ加符・待ち加符等）は適用しない。七対子の符は**常に25符固定**とする。これにより七対子の点数は以下となる：
> - 2翻25符: 子ロン1,600 / 親ロン2,400
> - 3翻25符（例: 七対子+タンヤオ）: 子ロン3,200 / 親ロン4,800
> - 4翻25符: 子ロン6,400 / 親ロン9,600

### 付録A: 符計算の詳細

| 項目 | 符 |
|---|---|
| 基本符（副底） | 20符 |
| メンゼンロン加符 | +10符 |
| ツモ加符 | +2符（平和ツモ除く） |
| **メンツ加符** | |
| 中張牌の明刻 | +2符 |
| 中張牌の暗刻 | +4符 |
| 么九牌の明刻 | +4符 |
| 么九牌の暗刻 | +8符 |
| 中張牌の明槓 | +8符 |
| 中張牌の暗槓 | +16符 |
| 么九牌の明槓 | +16符 |
| 么九牌の暗槓 | +32符 |
| **雀頭加符** | |
| 役牌の雀頭 | +2符 |
| **待ち加符** | |
| 辺張・嵌張・単騎 | +2符 |
| 両面・双碰 | +0符 |

> **【七対子の特例】**
> 七対子は上記の符計算ルールを一切適用せず、**常に25符固定**とする。
> 10符単位の切り上げも行わない（25符のまま計算）。

> **【平和ツモの特例】**
> 平和ツモの場合、ツモ加符(+2符)を加えず**20符固定**とする。

### 付録C: 赤ドラの構成

| 牌 | 通常枚数 | うち赤ドラ | 備考 |
|---|---|---|---|
| 五萬（5m） | 4枚 | **1枚** | 赤5萬×1 + 通常5萬×3 |
| 五筒（5p） | 4枚 | **2枚** | 赤5筒×2 + 通常5筒×2 |
| 五索（5s） | 4枚 | **1枚** | 赤5索×1 + 通常5索×3 |
| **赤ドラ合計** | | **4枚** | 赤ドラ1枚につき1翻加算 |

> **【赤ドラの計算ルール】**
> - 赤ドラは役ではなく、ドラと同様に翻を加算するボーナスである
> - 赤ドラだけでは和了できない（他に最低1つの役が必要）
> - 筒子の5は赤が2枚ある点に注意（一般的な日本麻雀のルール）
> - カメラ認識時は牌の色（赤/黒）を判別して赤ドラを識別する
